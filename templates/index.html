くずし字列分割アノテーションツール

概要

日本古典籍くずし字データセットの列分割アノテーションを行うツールです。各文字のbboxを列単位でグループ化し、列画像とCSVアノテーションを出力します。

基本操作

  1. 書籍とページを選択（起動時は自動で最初のページを表示）
  2. ↓キーで列の最後の文字まで移動
  3. 次の列の先頭文字でEnterキー → 直前までが列として確定
  4. 最後の列は、最後の文字でEnterキー → 残り全部が確定
  5. Sキーで保存 → CSVと列画像を出力

キーボードショートカット

↑ ↓文字選択を移動
← →前後のページへ移動
Enter列を確定（選択した文字から新しい列開始）
Z元に戻す (Undo)
Yやり直し (Redo)
S保存
Hこのヘルプを表示
V列/セグメント表示を切り替え
N選択文字を未分割に戻す
Shift + N全解除（全て未分割に戻す）
AAutoモード ON/OFF 切り替え
Enter (セグメント表示)セグメントを分割（選択列の位置で分割）
N (セグメント表示)セグメントを結合（前のセグメントと結合）
R (セグメント表示)Segment ID を変更

表示の見方

  • 未割り当ての文字
  • 選択中の文字
  • 確定済みの列（色で区別）
  • Char ID順の重心線

Autoモード

ONにすると、既存のアノテーションがないページを開いた時に、X座標に基づいて自動で列を推定します。推定結果を修正してから保存できます。

学習済みモデルがある場合は、Auto推定より先にモデル推論結果を初期値として読み込みます。優先順位は「書籍別モデル → 全書籍共通モデル → 従来Auto」です。

Human in the Loop

保存済みの列・セグメントから書籍ごとの境界モデルを学習し、新しいページではその推論結果を初期値として使えます。誤りは通常の列編集・セグメント編集で修正し、その保存結果が次の学習データになります。

右側パネルでは、現在の書籍に対する列/セグメントモデルの学習状況、学習時 metrics、直近の human feedback を確認できます。

表示モード

右側パネルのスイッチまたは V で「列」表示と「セグメント」表示を切り替えられます。セグメント表示中は、左に列表示・右にセグメント表示を並べて比較できます。

セグメント表示中は、左ペインをクリックすると列編集、右ペインをクリックするとセグメント編集に切り替わります。

セグメント表示中は Segment ID の編集・保存（S）ができます（列の保存後のみ）。

出力ファイル

  • output/{book_id}/column_annotation.csv - 列アノテーションCSV
  • output_seg/{book_id}/column_annotation.csv - セグメントアノテーションCSV（Segment ID追加）
  • output/{book_id}/columns/{page_id}_col0001.jpg - 列画像
